Project : MSU ROOMMATE MATCH SYSTEM

Overview

This project focuses on creating a full-stack web application designed to help university students find compatible roommates based on personal preferences and lifestyle habits. .

Features

  • User authentication and profile creation
  • Preference-based matching algorithm e.g., gender, noise tolerance, room sharing, etc.
  • Real-time match cards with compatibility scores
  • Accept/reject match system with mutual confirmation notifications
  • Profile image upload and editable preferences dashboard

Technologies Used

  • React
  • Node.js
  • MongoDB
  • HTML/CSS/JavaScript for frontend

Challenges Faced

Syncing data between React components and the Node/Express backend (especially for live updates like accept/reject flow) required careful state management and API design

Conclusion

RoommateMatch was a valuable project that deepened my understanding of full-stack development, especially in handling real-world user interaction flows and data-driven matching logic. It challenged me to think critically about user experience, security, and system design. This project not only strengthened my skills in React, Node.js, and MongoDB but also taught me how to build scalable and user-focused web applications.
